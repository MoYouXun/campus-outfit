# Campus Outfit 校园穿搭项目

## 项目概述

Campus Outfit是一个面向校园学生的穿搭分享社区平台，提供穿搭展示、互动交流、天气推荐等功能。项目采用前后端分离架构，前端使用Vue 3 + TypeScript + Tailwind CSS，后端使用Spring Boot + MyBatis-Plus + MySQL。

### 核心功能

#### 1. 控制器示例 (OutfitController.java)

```java
package com.campus.outfit.controller;

import com.campus.outfit.dto.OutfitAnalyzeRequest;
import com.campus.outfit.service.AiService;
import com.campus.outfit.service.WeatherService;
import com.campus.outfit.utils.Result;
import com.campus.outfit.vo.OutfitAnalyzeResponse;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/outfit")
public class OutfitController {

    @Autowired
    private AiService aiService;

    @Autowired
    private WeatherService weatherService;

    /**
     * 穿搭分析接口
     */
    @PostMapping("/analyze")
    public Result<OutfitAnalyzeResponse> analyzeOutfit(@RequestBody OutfitAnalyzeRequest request) {
        // 调用AI服务进行穿搭分析
        OutfitAnalyzeResponse response = aiService.analyzeOutfit(request);
        return Result.success(response);
    }

    /**
     * 获取天气信息
     */
    @GetMapping("/weather")
    public Result<?> getWeather(@RequestParam String city) {
        return Result.success(weatherService.getWeather(city));
    }
}
```

#### 3. 数据库初始化脚本 (schema.sql)

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS campus_outfit CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE campus_outfit;

-- 用户表
CREATE TABLE IF NOT EXISTS user (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    password VARCHAR(100) NOT NULL,
    avatar VARCHAR(255),
    nickname VARCHAR(50),
    gender TINYINT,
    role VARCHAR(20) DEFAULT 'USER',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    is_deleted TINYINT DEFAULT 0
);

-- 穿搭表
CREATE TABLE IF NOT EXISTS outfit (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    image_url VARCHAR(255) NOT NULL,
    description TEXT,
    tags VARCHAR(255),
    weather VARCHAR(20),
    occasion VARCHAR(50),
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    is_deleted TINYINT DEFAULT 0,
    FOREIGN KEY (user_id) REFERENCES user(id)
);

-- 互动表
CREATE TABLE IF NOT EXISTS interaction (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    outfit_id BIGINT NOT NULL,
    type VARCHAR(20) NOT NULL, -- LIKE, COLLECT, COMMENT
    content TEXT,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    is_deleted TINYINT DEFAULT 0,
    FOREIGN KEY (user_id) REFERENCES user(id),
    FOREIGN KEY (outfit_id) REFERENCES outfit(id)
);

-- 天气缓存表
CREATE TABLE IF NOT EXISTS weather_cache (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    city VARCHAR(50) NOT NULL,
    weather_data JSON NOT NULL,
    expire_time DATETIME NOT NULL,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## 功能特点

### 前端功能
1. **响应式设计**：适配不同屏幕尺寸
2. **瀑布流布局**：美观的穿搭卡片展示
3. **互动功能**：点赞、收藏等用户交互
4. **标签系统**：穿搭分类和筛选
5. **天气推荐**：根据天气提供穿搭建议

### 后端功能
1. **RESTful API**：规范的接口设计
2. **数据持久化**：MySQL数据库存储
3. **缓存机制**：Redis缓存热点数据
4. **文件存储**：MinIO存储用户上传的图片
5. **AI分析**：模拟AI穿搭分析功能
6. **异常处理**：全局异常处理机制
